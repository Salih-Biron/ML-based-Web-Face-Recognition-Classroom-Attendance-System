# 基于机器学习的 Web 人脸识别班级智能考勤系统

> **提示词文档**
> 创建人：曾绍庚
> 更新时间：2026.06.15 14:45

---

## 一、项目概述

将当前项目重构为一个适用于 **Windows/Linux CPU 环境**，支持 **50~200 人规模**的基于机器学习的 Web 人脸识别班级智能考勤系统。

- 考勤无固定周期，时间灵活，工作日与周末均可发起
- 采用 **C/S 架构**，服务端基于 Python + Flask，客户端基于 JavaScript

---

## 二、服务端架构

### 2.1 人脸识别技术栈

```
DeepFace (ArcFace)
        ↓
Embedding (L2 归一化)
        ↓
Cosine Similarity
        ↓
Threshold（默认 0.22）
        ↓
Known / Unknown（已知人员 / 未知人员）
```

### 2.2 运行环境要求

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10/11 或 Linux |
| Python 版本 | 3.10（推荐） |
| 硬件 | CPU 即可运行（无需 GPU） |
| Web 框架 | Flask（实现与客户端的 Web 通讯） |

### 2.3 人脸库目录结构

在项目根目录创建 `face_db/` 文件夹，按以下结构放置人脸照片：

```
face_db/
├── 教育技术/
│   ├── 张三1.jpg
│   ├── 李四1.jpg
│   └── 王五1.jpg
├── 大数据/
│   ├── 张三2.jpg
│   ├── 李四2.jpg
│   └── 王五2.jpg
└── 人工智能/
    ├── 张三3.jpg
    ├── 李四3.jpg
    └── 王五3.jpg
```

- **第一层文件夹**：班级名称
- **第二层文件**：人员照片，文件名即人员姓名
- **支持图片格式**：`jpg`、`jpeg`、`png`、`bmp`、`webp`
- 运行时可选择一个或多个或全选班级文件夹进行训练和识别

## 三、客户端系统

客户端分为**教师用户管理页**和**学生用户签到页**两种身份，均基于 JavaScript 实现。

---

### 3.1 教师用户管理页

#### 3.1.1 登录与权限

- 需要登录，默认账号：`admin` / 密码：`admin123`
- 暂不开放注册功能

#### 3.1.2 活动管理

- 支持考勤活动的**增删改查**
- 新增、删除活动，或修改活动的起止时间后，可根据服务器当前 IP 地址生成**学生端局域网签到网址**
- 可选将签到网址生成**考勤活动二维码**，方便学生手机扫码签到（学生签到页需适配电脑和手机主流分辨率）
- 活动状态根据活动结束时间**自动刷新**

#### 3.1.3 班级管理

- 支持查看、增加、删除、修改班级列表
- 支持**单选、多选或全选**班级发布考勤活动
- 系统根据用户选择的班级，自动加载或构建人脸库：
  - 若服务端已存在对应班级名字的**人脸库缓存文件**，则自动加载
  - 否则直接构建人脸库，批量提取人脸特征并按班级名称缓存为文件
  - 多选班级时，将选中班级名称按默认排序组合作为缓存文件名

#### 3.1.4 人员管理

- 支持按班级**查看、改名、添加、删除**人员信息
- 添加人员时可通过**单张或多张照片**导入到用户选择的班级文件夹
- 人员发生变动时，自动删除该班级对应的所有人脸库缓存文件，下次加载时重新构建

#### 3.1.5 时长统计与排名

- 可查看学生参与活动的**本日、本周、本月、本学期**总时长实时排名
- 学期的开始和结束日期可在教师端**设置与修改**

#### 3.1.6 数据库查看

- 教师端可选择某个班级，列出所有人员信息

---

### 3.2 学生用户签到页

#### 3.2.1 操作按钮

页面提供三个操作按钮，可切换当前考勤状态：

| 按钮 | 功能 |
|------|------|
| 签到 | 开启摄像头，开始签到采集 |
| 签退 | 开启摄像头，开始签退采集 |
| 关闭 | 关闭摄像头，停止采集 |

#### 3.2.2 摄像头实时识别

- 点击"签到"或"签退"按钮时，调用用户摄像头
- 每隔 **3 秒**截取一帧当前摄像头画面，提交至后台人脸识别接口
- 支持识别**单帧图片中的所有人脸**
- 确保摄像头调用符合浏览器安全规范，处理好用户授权流程

#### 3.2.3 识别信息显示

在摄像头实时识别窗口下方显示识别结果，示例格式：

```
检测到人脸：1，成功标记：1，未识别/低分：0
zsg_0004、ldh_0002 签退成功。
```

提示文本根据实际识别到的用户信息动态生成。

#### 3.2.4 重复签到与签退逻辑

| 情况 | 处理逻辑 |
|------|----------|
| 已签到人员再次签到 | **不提示"已签到，未重复记录"**，直接更新该人员签到时间为最新时间 |
| 未签到人员进行签退 | 显示提示：`${人员标识}未签到，此次签退无效`（如：`zsg_0004未签到，此次签退无效`） |
| 正常签退 | 状态设置为"已签退" |

#### 3.2.5 签到人员信息展示

在识别信息显示窗口下方，按签到时间从小到大排列显示：

- 🟢 **绿色字体**：已签到的人员信息 + "已签到" + 签到时间
- 🔵 **蓝色字体**：已签退的人员信息 + "已签退" + 签退时间
- 🔴 **红色字体**：未签到的人员信息 + "未签到"

#### 3.2.6 时长排名（实时）

- 在签到人员信息展示窗口下方显示排名信息
- 支持按**日、周、月、学期**四种时间维度切换查看
- 时长统计规则：
  - 根据每人每次签退和签到时间间隔，累加换算为小时
  - 人员状态为"未签到"时，不记录时长；需等状态变为"已签到"后才开始计时
  - 多次签到和签退的时长累计叠加
- 自动清零规则：

| 时间节点 | 清零操作 |
|----------|----------|
| 每日 0:00 | 所有人员签到信息重置为"未签到"，当日统计时长清零 |
| 每周一 0:00 | 所有人员本周统计时长清零 |
| 每月 1 日 0:00 | 所有人员本月统计时长清零 |
| 学期开始时间（可配置） | 所有人员本学期统计时长清零 |

## 四、附录

### 4.1 关键技术约束

- 服务端采用 DeepFace + ArcFace + Cosine Similarity 轻量级人脸识别方案，适配 CPU 环境
- Cosine Similarity 默认阈值为 **0.22**，可按需调整
- 人脸库缓存文件命名规则：多选班级时按默认排序组合班级名称

### 4.2 接口安全要求

- 图像采集、传输及接口调用过程需保证稳定性和安全性
- 摄像头调用需遵循浏览器安全规范（建议使用 HTTPS 或 localhost）
- 处理好用户摄像头授权流程
