# 基于机器学习的 Web 人脸识别班级智能考勤系统

一个基于深度学习的人脸识别考勤系统，支持多人同时签到，适用于教室考勤场景。

## 项目简介

本系统采用 C/S 架构，使用 DeepFace + ArcFace 进行人脸识别，支持 50-200 人规模的班级考勤管理。教师可通过管理端发布考勤活动并生成签到链接/二维码，学生通过手机或电脑浏览器访问链接，使用摄像头完成签到签退。

### 核心特性

- 🎯 **多人同时识别**：单帧图像支持识别多张人脸（5人+）
- 📱 **移动端友好**：支持手机扫码签到，响应式界面
- 🔄 **实时反馈**：智能提示识别结果，未识别人员可调整位置重试
- ⚡ **CPU优化**：基于缓存机制优化性能，无需GPU即可运行
- 📊 **统计分析**：支持日/周/月/学期时长统计与排名
- 🌐 **局域网部署**：教师电脑作为服务器，学生通过WiFi访问

## 技术栈

## 安装部署

### 环境要求

- Python 3.10+
- Windows 10/11 或 Linux
- 至少 2GB RAM
- 摄像头设备

### 安装步骤

1. **克隆项目**
```bash
git clone https://github.com/Salih-Biron/ML-based-Web-Face-Recognition-Classroom-Attendance-System.git
cd ML-based-Web-Face-Recognition-Classroom-Attendance-System
```

2. **安装依赖**

Windows环境：
```bash
python -m pip install -r requirements.txt
```

Linux/Mac环境：
```bash
pip install -r requirements.txt
```

3. **初始化数据库**
```bash
python -c "from modules.database import init_db; init_db()"
```

4. **创建人脸库目录**
```bash
mkdir face_db face_embeddings_cache
```

5. **准备人脸数据**

在 `face_db/` 目录下按班级创建文件夹，并放入学生照片：
```
face_db/
├── 教育技术/
│   ├── 张三.jpg
│   └── 李四.jpg
├── 大数据/
└── 人工智能/
```

**照片要求**：
- 格式：jpg、jpeg、png、bmp、webp
- 文件名即为学生姓名
- 建议正面清晰照片

6. **启动服务**
```bash
python app.py
```

服务启动后：
- 教师端：`http://localhost:5000/teacher/login`
- 学生端：`http://localhost:5000/student/<activity_id>`

### 局域网访问

修改 `app.py` 启动配置：
```python
if __name__ == '__main__':
    app.run(host='0.0.0.0', port=5000, debug=True)
```

获取本机IP地址（Windows）：
```bash
ipconfig
```

学生通过 `http://<教师电脑IP>:5000/student/<activity_id>` 访问

## 使用说明

### 教师端操作流程

1. **登录系统**
   - 访问 `http://localhost:5000/teacher/login`
   - 使用默认账户：admin / admin123

2. **添加班级**
   - 进入"班级管理"标签
   - 点击"新增班级"，输入班级名称

3. **添加学生**
   - 进入"人员管理"标签
   - 选择班级，上传学生照片（支持批量）

4. **创建考勤活动**
   - 进入"活动管理"标签
   - 点击"新增活动"
   - 填写活动名称、开始/结束时间
   - 选择参与班级（支持多选）
   - 生成签到链接和二维码

5. **查看统计**
   - 进入"时长统计"标签
   - 选择时间维度（日/周/月/学期）
   - 查看学生考勤时长排名

### 学生端操作流程

1. **访问签到页面**
   - 扫描二维码或打开签到链接
   - 确保设备在同一WiFi下

2. **启动摄像头**
   - 点击"启动摄像头"按钮
   - 允许浏览器调用摄像头权限

3. **开始签到**
   - 点击"开始签到"按钮
   - 面对镜头，等待识别（每3秒自动捕获识别）
   - 系统显示识别结果和置信度
   - 支持多人同时识别

4. **签退**
   - 已签到的学生再次面对镜头
   - 系统自动识别并完成签退
   - 自动计算考勤时长并更新统计

5. **查看考勤列表**
   - 页面下方实时显示考勤记录
   - 自动每5秒刷新
   - 按状态排序（已签退 > 已签到 > 未签到）

## 核心机制

### 人脸识别流程

1. **人脸库加载**：根据选择的班级加载或构建人脸特征库
2. **缓存机制**：提取的人脸特征向量缓存为 `.pkl` 文件，避免重复计算
3. **多人识别**：使用 DeepFace 检测单帧图像中的所有人脸
4. **相似度计算**：计算余弦相似度，阈值默认 0.22
5. **结果返回**：返回识别成功的学生信息

### 签到签退逻辑

- **未签到 → 签到**：首次识别成功，记录签到时间，状态变为"已签到"
- **已签到 → 签退**：再次识别成功，记录签退时间，计算时长（分钟），累加到统计数据
- **已签退 → 已完成**：第三次识别提示"已完成"，不再更新
- **自动识别**：每3秒自动捕获并识别，无需手动操作
- **多人支持**：单次识别可同时处理多张人脸，每人独立计算状态

### 自动重置机制

- **每日 0:00**：重置所有人签到状态，清零当日时长
- **每周一 0:00**：清零本周时长
- **每月 1 日 0:00**：清零本月时长
- **学期开始**：清零学期时长

### 常见问题

### Q1: pip命令无法识别？

**Windows环境解决方案**：
```bash
# 方法1：使用python -m pip
python -m pip install -r requirements.txt

# 方法2：添加pip到PATH（需要管理员权限）
# 找到Python安装目录下的Scripts文件夹，添加到系统环境变量

# 方法3：使用Python完整路径
C:\Users\你的用户名\AppData\Local\Programs\Python\Python310\python.exe -m pip install -r requirements.txt
```

### Q2: 摄像头无法调用？

**原因**：部分浏览器要求 HTTPS 才能调用摄像头

**解决方案**：
- 使用 `localhost` 访问（开发环境）
- 或配置 HTTPS 证书（生产环境）

### Q3: 识别速度慢？

**原因**：CPU 环境下，大规模人脸库识别需要时间

**优化方案**：
- 系统已实现人脸特征缓存
- 首次加载较慢，后续识别速度会提升
- 建议人脸库控制在 50-200 人规模

### Q4: 识别率低？

**原因**：光线、角度、距离影响识别效果

**改善方法**：
- 确保光线充足
- 正面面对摄像头
- 保持适当距离（0.5-1米）
- 调整相似度阈值（系统设置中）

### Q5: 多人同时签到识别不全？

**原因**：人脸遮挡或距离过远

**解决方案**：
- 调整站位，避免遮挡
- 靠近摄像头
- 系统每3秒自动识别，未识别的人可等待下一次

## 性能与限制

### 性能指标
- 支持人脸库：50-200 人
- 单次识别耗时：2-5 秒（CPU环境）
- 多人识别：支持单帧 5+ 人脸

### 已知限制
- CPU性能受硬件影响，大规模人脸库识别较慢
- 弱光环境识别率下降
- 部分浏览器摄像头API兼容性问题
